# Fabula Alterius
テキストベースの軽量なシングルプレイブラウザRPGです。
「データ駆動設計」と「堅牢なCIパイプライン」を軸に、個人開発におけるコード品質の維持と拡張性を追求しています。

## アーキテクチャ
データ駆動設計により高い拡張性を持つ

## 使用技術
フロントエンド
- React: メジャーなライブラリを選択することで、トラブルシューティングの効率化を図っています。
- Tailwind CSS: スタイリングの迅速化と、CSSの肥大化防止。
- Zustand / Zod: ゲーム状態のグローバル管理と、スキーマ定義によるデータの整合性確保。
- React Hook Form: 入力バリデーションの簡略化。
開発・品質管理 (DX & QA)
- Vite / Biome: 高速なビルド環境と、ツール間のコンフリクトを排除したモダンなLinter/Formatter環境。
- Vitest / React Testing Library / Playwright: 単体テストからE2Eテストまでをカバーし、リファクタリング耐性を確保。
- Husky & Lint-staged: ローカル環境での「雑なコード」のコミットを未然に防ぐ仕組み。
- GitHub Actions: 継続的インテグレーション（CI）により、常にテストと型チェックをパスした状態を維持。
### 技術的な意思決定(採用担当の方へ)
1.本プロジェクトでは、SPA（Single Page Application）でありながらルーター系ライブラリを意図的に導入していません。
- 理由: URLによる状態遷移を許容すると、「未到達エリアへの直接アクセス」や「ブラウザの戻るボタンによる戦闘結果のロールバック（不正操作）」を容易にする脆弱性につながると判断したためです。
- 対策: ゲーム内の進行フラグに基づき、コンポーネントレベルで厳密なステート制御を行っています。

2.データ駆動設計による拡張性
ゲーム内のイベントやアイテムデータは、ロジックから分離されたデータ層として定義しています。これにより、コードを修正することなくコンテンツを追加できる拡張性を備えています。

3.Biome の採用
ESLint と Prettier の併用による設定の複雑化やコンフリクトを避けるため、Rust製の高速な統合ツールである Biome を採用しました。
「ツールのための設定」を最小限にし、開発効率を最大化する選択をしています。
